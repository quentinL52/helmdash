## 2024-06-11 - Cache Utility Functions & Avoid Redundant Sorting
**Learning:** `getMonthlyEntries` inside `src/lib/finance-utils.ts` already returns an array sorted in descending order by month. Several components were found explicitly spreading and re-sorting its output `[...getMonthlyEntries(finance.entries)].sort(...)` during every render, which is an unnecessary overhead, especially for operations handling financial time series. Furthermore, `getMonthlyEntries` performs a full pass of all items (O(N) operation to group items by month) which was being invoked multiple times redundantly within the same component (like `runway-chart.tsx` and `finance-charts.tsx`).
**Action:** When working with utility functions that already return sorted/grouped data, avoid re-sorting. Always cache the result of iterative utility functions in a local variable or memoized hook (like `useMemo`) when they need to be accessed multiple times within the same component cycle.
