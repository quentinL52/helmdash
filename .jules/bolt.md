## 2024-05-30 - [Performance Anti-Pattern: Redundant Sorting of Pre-Sorted Collections]
**Learning:** `getMonthlyEntries` naturally returns an array sorted in descending order by month. However, many components were destructuring and re-sorting the result using `[...getMonthlyEntries(...)].sort(...)`, causing unnecessary O(n log n) overhead on every render/memoization trigger and needlessly duplicating the array. Additionally, calling it repeatedly across components, and inside loops (like in `finance-charts.tsx`), severely impacted performance.
**Action:** Always verify if helper functions return pre-sorted data. Cache expensive utility functions in a local scope variable within `useMemo` hooks rather than making duplicate inline calls, particularly when passing identical state references.
