## 2025-02-14 - Fix missing authentication in cron process-scheduled endpoint
**Vulnerability:** The `/api/cron/process-scheduled` endpoint in `src/app/api/cron/process-scheduled/route.ts` was missing the `req: Request` parameter in its `GET` handler signature, causing it to fail to retrieve and verify the `Authorization` header against the `CRON_SECRET`. This left the endpoint entirely unauthenticated, allowing arbitrary invocation of scheduled task processing and automated account deletion routines.
**Learning:** In Next.js App Router API routes, handlers must explicitly declare the `req: Request` argument if they intend to read headers. The fallback comment correctly noted that headers weren't accessible, but rather than bypassing the check, the handler signature needed to be fixed. Additionally, endpoints relying on a secret (like `CRON_SECRET`) must verify the secret is both defined and matched to fail securely.
**Prevention:** Always verify that API route handlers explicitly receive `req: Request` when header-based authentication is required, and ensure that authorization checks explicitly fail if the required environment secrets are undefined.
