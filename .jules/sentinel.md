## 2024-05-18 - Missing Authentication in Cron API Route
**Vulnerability:** The `/api/cron/process-scheduled` Next.js endpoint was entirely unauthenticated because it was missing the `req: Request` parameter in its signature. It also did not check if `CRON_SECRET` was defined, and other cron routes didn't check for undefined `CRON_SECRET` either.
**Learning:** In Next.js App Router, omitting `req: Request` from an API route handler prevents access to request headers, which can lead to accidental bypassing of header-based authentication logic (like cron secrets). Additionally, comparing against `undefined` in template strings (e.g. `` !== `Bearer ${process.env.CRON_SECRET}` ``) can be bypassed by sending `Authorization: Bearer undefined` if the environment variable is missing.
**Prevention:** Always explicitly define `req: Request` when header validation is required, and explicitly check that security secrets are defined (truthy) before attempting to validate them against incoming headers.
